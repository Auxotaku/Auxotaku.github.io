初次搭建过程记录

发表于2023-03-07|更新于2023-03-07
|阅读量:

总的来说，得益于hexo框架的清晰与自动部署插件的强大，这次搭建从技术上十分简单

但是

由于不稳定的机场和较为生疏的git远程操作，这次搭建也遇到了不少问题

那么就把搭建过程作为第一篇blog的内容吧

hexo

说实话，hexo的官方文档看的出是“历史悠久”，下方评论区对于文档的抱怨是年年都有

由于想尝试一下正派的部署过程，我还是决定从官方文档开始从零搭建文档 | Hexo （中文支持好评）

安装

由于我的win本的git和key都配置好了 所以仅仅需要安装node.js

选择官网LTS版本的安装包（话说为啥是直接用安装包安装，真方便

注意一下配置PATH之后即可直接使用hexo命令，要不然每次还要npm hexo

搭建

hexo 的 默认主题完全不丑

但是不折腾我还建站干嘛（

butterfly , yes！

由于是在已有框架内进行搭建

我其实只是精装修一下罢了，再套上butterfly的主题，这基本上是拎包入住了

配置

所以其实搞懂如何配置就可以轻松搭建了

但是，对于不按顺序看文档的Aux来说，一开始对于config其实非常迷茫

总的来说

就是*__config*文件负责网站配置 scaffolds内的Markdown文件作为模板

而文章等内容的格式 是在source中每篇内容Markdown文档最前分隔符内进行配置

主题

单拿主题出来说一下

https://butterfly.js.org/ 中的文档清晰

但是对于第一次配置的我来说，有一个关键问题

我咋刷新到这个新主题呢

这就是不好好按顺序看文档的下场（

简单来说 就是先生成、再推送

如果你只是想看一下你的页面现在长什么样子

你只需启动一下serve 在localhost看一下你的静态页 hexo s

部署

主要的坑都踩在部署上

如果使用Github Pages进行部署

那么其实你的配置过程非常简单，在_config文件中 在deploy配置中设置好你的项目地址即可（谁敢信我第一次把地址写错了，debug半天

但是考虑到网络问题，试一试gitee也未尝不可

不得不推荐一下一键部署功能，省去了很多麻烦，也很符合我心中对个人简易博客的想象

安装npm install hexo-deployer-git --save

在配置好后就可以推送

当然，如果不使用ssh密钥认证还得进行用户名密码的输入

在一个个坑都踩掉后

就能看到你的网页了

遇到的主要问题

1、不稳定的网络状况 得换个好点的机场了 48一年也太不稳定了

2、别把仓库名写错

3、在尝试不使用自动部署时，将已有文件夹推送到远端仓出现了报错src refspec main does not match any

这主要是由于GitHub目前默认的分支main与本地默认分支master的差异

（GitHub使用main的原因居然是master有些Slavery 太乐了

end

本次配置耗时约3h

与代码能力与计算机网络相关知识没有一点关系（我还想借着学学计网呢

感觉甚至可以给非CS专业学生食用
